# Knowledge Bases Registry

## Purpose

This file is the registry of knowledge bases in the current `4DreamTeam` workspace.

Instructions for using `4DreamTeam` are in the root `README.md`.

## Knowledge Bases

| Project | Type | Status |
|---|---|---|
| `<project-name>` | `backend / frontend / fullstack / library/SDK / mixed / unknown` | `proposed / actual / accepted / superseded / deprecated / unknown` |

## Rules

1. Each knowledge base lives in `docs/<project-name>`.
2. Each knowledge base must have its own `sources.md`.
3. Project-specific architecture, domains, flows, contracts, schemas, and ADRs live only inside the project folder.
4. New knowledge bases are added to this registry after `wiki bootstrap` or `wiki blueprint`.
5. This file must not contain project architecture, domain model, or technical details for a specific project.
