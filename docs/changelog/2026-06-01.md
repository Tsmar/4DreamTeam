# Changelog 2026-06-01

## Content

## Unreleased

- Added `4dt-web` and the separate `packages/web` Workspace View server for local, read-only browsing of managed workspace knowledge in a browser.
- Added Workspace View wiki search, page navigation, related links, tag pages, tag index, fixed header, responsive sidebar, mobile burger navigation, automatic header search, and light/dark theme support with system preference defaults.
- Taught workspace instructions and installed runtime validation about launching Workspace View as the local web panel at `http://localhost:4174/`.
- Reorganized managed wiki information architecture around path-based pages, breadcrumbs, alphabetic sidebar grouping, and clearer `start/`, `product/`, `workflows/`, `architecture/`, `domains/`, `schemas/`, and `contracts/` boundaries.
- Moved exported workflow docs from `docs/flows/` to `docs/workflows/`, moved the workspace overview to `docs/start/overview.md`, and updated exported wiki links and page metadata accordingly.
- Adopted dated source changelog files: `CHANGELOG.md` is now an index, while `changelog/YYYY-MM-DD.md` files hold all source-history entries for a single day.
- Mirrored dated changelog history into managed wiki pages under `changelog/YYYY-MM-DD.md` so Workspace View can browse history page by page.
- Updated release instructions and the release plan template to target the daily changelog file plus the root index.
- Updated wiki write guidance for SQLite-backed storage: parallel reads/searches and independent page writes are allowed, while same-page logical updates should be combined into one `page apply` payload.
- Trimmed tests that asserted legacy file-storage absence and adjusted validation coverage for the SQLite-backed tool storage model.
- Documented the canonical board column and status contract in workspace instructions and role references, exposed status lists through `4dt-board types list`, and made invalid task or epic statuses fail immediately on board metadata updates.

## 0.5.8

- Added per-domain schema recording in shared `.4dt/db.sqlite3` table `tool_schema_versions`, with schema version, schema hash, tool version, and applied timestamp for board, wiki, sources, search, and memory.
- Added `4dt-db schema status` so agents can inspect recorded schema state without manually comparing SQL first.
- Changed `4dt-memory` schema setup to create only the current packaged schema and report `schema_mismatch` for older or incomplete memory schemas instead of silently migrating them.
- Added operator-controlled schema upgrade guidance: backup first, create a clean current schema, compare recorded domain schema versions and hashes, and ask for operator approval before migration or reset.
- Added `4dt-db migrate plan/apply` for declarative SQL-side data movement without exposing row contents to agents.
- Added migration operations for column-to-column table copy, default values, table rename/drop with explicit `--allow-drop`, count validation, not-null validation, foreign-key validation, and schema registry updates.
- Stopped `4dt-board validate` from creating legacy `.4dt/board/tasks` directories now that board state is SQLite-authoritative.
- Added regression coverage for startup schema bootstrap without legacy domain directories across board, wiki, sources, search, and memory.
- Bumped the skill metadata and documented README version to `0.5.8`.
